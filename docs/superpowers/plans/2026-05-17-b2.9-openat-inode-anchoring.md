# B2.9 openat/fchdir inode-anchoring — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `openat(dirfd,…)`/`fchdir`/`getcwd` POSIX-faithful by anchoring directory fds and `Process.cwd` to a directory **inode** instead of a path snapshot, so they stay correct after the directory is renamed (and fail `ENOENT` after it is removed).

**Architecture:** A default-`None` `VfsBackend` dir-handle API (`dir_inode`/`resolve_at`/`dir_path`); `RamfsBackend` gains a directory-inode model + recursive subtree re-key on directory rename; `FdEntry::Directory` and `Process.cwd` become a dual `(mount_id, dir_inode: Option<u64>, path)` shape; `sys_openat` does an inode-anchored component walk that re-delegates to the existing path-based `sys_open` on `.`/`..`, mount boundaries, and symlinks; `PathResolver` refreshes `cwd.path` from the live `dir_path` before every relative resolution. Backends without dir inodes (hostfs, overlay-if-deferred) run a tracked degraded mode.

**Tech Stack:** Rust (`packages/kernel-wasm`, `wasm32-wasip1`), `cargo test` (native), deno parity (`wasm-kernel-imports_test.ts`), B0 differ + `abi/conformance` + `abi/conformance/parity-baseline.toml`.

**Spec:** `docs/superpowers/specs/2026-05-17-b2.9-openat-inode-anchoring-design.md` (approved; pass-3 final). Branch `parity-b2.9-openat-inode-anchor`, worktree `.worktrees/b2.9-openat`, rebased on `main` @ `26406de`.

    pub fn dir_inode_at(&self, path: &[u8]) -> Option<(MountId, u64)> {

- [ ] **Step 3: Implement recursive re-key.** In the directory branch of `RamfsBackend::rename`, after validating, replace the `dirs.remove/insert` with a prefix re-key over `paths`, `symlinks`, `dir_inodes`, `dir_paths`:

```rust
        // Directory rename: re-key every descendant from old prefix
        // to new prefix. Inode ids are preserved (only path keys
        // move) so open file OFDs (which hold `inode`, not path) and
        // every dir inode survive unchanged.
        let with_slash = |p: &[u8]| {
            let mut v = old_path.to_vec();
            v.push(b'/');
            p.starts_with(&v) || p == old_path
        };
        let remap = |p: &[u8]| {
            let mut np = new_path.to_vec();
            np.extend_from_slice(&p[old_path.len()..]);
            np
        };
        // paths (files)
        let moved: Vec<(Vec<u8>, u64)> = self
            .paths
            .iter()
            .filter(|(k, _)| with_slash(k))
            .map(|(k, v)| (k.clone(), *v))
            .collect();
        for (k, v) in moved {
            self.paths.remove(&k);
            self.paths.insert(remap(&k), v);
        }
        // symlinks
        let moved: Vec<(Vec<u8>, Vec<u8>)> = self
            .symlinks
            .iter()
            .filter(|(k, _)| with_slash(k))
            .map(|(k, v)| (k.clone(), v.clone()))
            .collect();
        for (k, v) in moved {
            self.symlinks.remove(&k);
            self.symlinks.insert(remap(&k), v);
        }
        // dir inodes (forward + reverse), preserving ids
        let moved: Vec<(Vec<u8>, u64)> = self
            .dir_inodes
            .iter()
            .filter(|(k, _)| with_slash(k))
            .map(|(k, v)| (k.clone(), *v))
            .collect();
        for (k, id) in moved {
            self.dir_inodes.remove(&k);
            let nk = remap(&k);
            self.dir_inodes.insert(nk.clone(), id);
            self.dir_paths.insert(id, nk);
        }
        0
```

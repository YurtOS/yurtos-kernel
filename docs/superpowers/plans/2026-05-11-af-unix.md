# AF_UNIX ‚Äî Full POSIX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:subagent-driven-development` (recommended) or
> `superpowers:executing-plans` to implement this plan task-by-task. Steps
> use checkbox (`- [ ]`) syntax for tracking.

**Spec:**
[2026-05-11 AF_UNIX Design](../specs/2026-05-11-af-unix-design.md)

**Goal:** Replace PR #22's TCP-loopback `socketpair()` emulation with a
proper in-kernel AF_UNIX registry that covers the full POSIX surface ‚Äî
pathname and abstract addresses, SOCK_STREAM and SOCK_DGRAM, SCM_RIGHTS
fd passing, SO_PEERCRED peer credentials ‚Äî observably identical on
Wasmtime, Deno/Node, and browsers.

**Non-goals for this initiative:**

- Host AF_UNIX bridging (connecting guest sockets to real Unix sockets
  on the host). Separate embedder opt-in, future work.
- `SO_PASSCRED` / `SCM_CREDENTIALS` (in-band credential passing). SO_PEERCRED
  covers the common case.
- `SOCK_SEQPACKET`. Add when a port asks.
- Filesystem mode-bit gating of `connect()`. Embedder policy is the gate.

## Reusable infrastructure (no changes needed)

From Phase 1 exploration:

- `ListenerRegistry` paired-socket model extends to AF_UNIX by adding
  a second key namespace; existing waiter / rx-queue / async-sync
  machinery carries over.
- `SocketBackend` interface is already protocol-agnostic (no host/port
  in the type signature) ‚Äî just adds optional `path` on requests.
- `FdTarget.refs` refcounting is already correct for SCM_RIGHTS-style
  fd sharing; `closeFd()` decrements and only calls backend close when
  refs hit zero.
- `buildFdTableForSpawn/Fork`
  (`packages/kernel/src/process/kernel.ts:512-587`) ‚Äî the cross-pid
  fd-duplication pattern SCM_RIGHTS needs is already implemented for
  spawn/fork inheritance.
- `BrowserNetworkBridge` shared-registry pattern ‚Äî AF_UNIX works in
  browser for free.
- `authorizeListen()` policy hook ‚Äî extension point for new policy
  fields.
